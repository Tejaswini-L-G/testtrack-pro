# Frequently Asked Questions (FAQ)

## 1. How do I create a test case?

Navigate to the **Test Cases** section and click **Create Test Case**.  
Fill in the required fields such as title, description, priority, and expected result.

---

## 2. How do I execute a test case?

1. Go to **Execution Dashboard**
2. Select the test case
3. Mark the result as **Pass** or **Fail**

---

## 3. How do I report a bug?

When a test case fails during execution:

1. Click **Report Bug**
2. Enter bug details
3. Attach screenshots if needed
4. Submit the report

---

## 4. How do developers fix bugs?

Developers open the **Assigned Issues** page:

1. Click **Start Work**
2. Fix the issue in code
3. Provide **Fix Notes**
4. Add **Commit Link**
5. Mark bug as **Fixed**

---

## 5. How does bug verification work?

After a developer marks a bug as **Fixed**, the tester:

1. Re-tests the issue
2. Marks the bug as **Verified** if resolved
3. Otherwise marks it as **Reopened**

---

## 6. Can I export reports?

Yes.

The system allows exporting:

- Bug Reports
- Execution Reports
- Developer Performance Reports

in **PDF or Excel format**.

---

## 7. What integrations are supported?

TestTrack Pro supports:

- Git commit linking for bugs
- Webhooks for external systems
- REST API access with API keys
